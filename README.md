# Olist E-commerce Analytics Dashboard

Интерактивный аналитический дашборд для исследования данных бразильского маркетплейса Olist. Проект демонстрирует полный цикл работы с данными: от глубокого исследовательского анализа (EDA) до разработки масштабируемого BI-решения.

## 📈 Этапы исследования
Проект прошел путь от «сырых» данных к конкретным бизнес-инсайтам:
* **EDA (`01_eda_olist_data`)**: Первичная очистка данных, валидация связей между таблицами и понимание архитектуры маркетплейса.
* **Финансовый блок**: Анализ сезонности выручки, динамики объема заказов и концентрации спроса в ключевых регионах.
* **Логистический блок**: Мониторинг времени доставки, выявление «узких мест» в цепочках поставок и оценка надежности сервиса (метрика P90).
* **Клиентская аналитика (Retention)**: Анализ жизненного цикла клиентов, расчет повторных покупок и оценка лояльности аудитории.

## 🛠 Архитектура и технологии
- **Гибридный движок**: Код автоматически переключается между `PostgreSQL` (для локальной аналитики) и `SQLite` (в оперативной памяти для облачного деплоя). Это позволяет сохранять всю мощь SQL-запросов без необходимости настраивать сложную базу данных на сервере.
- **Интерактивность**: Реализована на базе `Streamlit`, позволяя пользователю взаимодействовать с данными в реальном времени.

## 📦 Стек технологий
- **Backend/ETL**: Python, Pandas, SQLAlchemy.
- **Database**: PostgreSQL (prod), SQLite (demo).
- **Frontend**: Streamlit, Seaborn, Matplotlib.

## 📋 Как запустить проект
1. **Клонировать репозиторий**:
   ```bash
   git clone [https://github.com/Rindikt/olist-ecommerce-analytics.git](https://github.com/Rindikt/olist-ecommerce-analytics.git)```
2. Установить зависимости: `pip install -r requirements.txt`
3. Запустить ETL (для локальной работы с БД): `python load_data.py`
4. Запустить дашборд: `streamlit run app.py`
